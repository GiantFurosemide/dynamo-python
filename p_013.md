# p_013: 审计意见 f_012.md 对应修改规划

依据审查报告 [f_012.md](f_012.md)，将必须修改项与建议修改项整理为可执行的修改规划，并按优先级实施。

---

## 一、必须修改（P0）

### 1.1 Classification 返回值统一为三元组

**依据**：f_012 §1 — 调用方统一按 `(best_ref, best_row, tr)` 解包，部分分支仅返回两值导致 `ValueError: not enough values to unpack (expected 3, got 2)`。

**文件**：`pydynamo/src/pydynamo/commands/classification.py`

| 位置 | 当前 | 修改后 |
|------|------|--------|
| `_classification_cpu_worker`：load 异常 return | `return 0, None` | `return 0, None, None` |
| `_classification_cpu_worker`：shape 不匹配 return | `return 0, None` | `return 0, None, None` |
| `_align_particle_task`：load 异常 return | `return None, None` | `return None, None, None` |
| `_align_particle_task`：shape 不匹配 return | `return None, None` | `return None, None, None` |

**验收**：`_classification_cpu_worker` 与 `_align_particle_task` 所有 return 均为 `(best_ref, best_row, tr)`；失败/跳过时 `tr=None`。`pytest pydynamo/test/test_classification.py -v` 全部通过。

---

### 1.2 Classification worker 文档与实现一致

**依据**：f_012 §2 — docstring 写 “Returns (best_ref, best_row) or (0, None)”，实际为 `(best_ref, best_row, tr)` / `(0, None, None)`。

**文件**：`pydynamo/src/pydynamo/commands/classification.py`，`_classification_cpu_worker` 的 docstring（约 50–53 行）。

**修改**：将 docstring 改为与实现一致，例如：  
“Returns (best_ref, best_row, tr). On load failure or shape mismatch returns (0, None, None). `tr` is the transformed volume for accumulation or None.”

**验收**：文档描述与所有 return 路径一致，并明确说明 `tr` 含义。

---

### 1.3 Alignment 默认配置去除本机绝对路径

**依据**：f_012 §3 — `alignment_defaults.yaml` 中 `nmask` 为 `/home/muwang/.../mask.mrc`，默认配置应可移植。

**文件**：`pydynamo/config/alignment_defaults.yaml`

**修改**：将 `nmask` 改为 `null`，并保留注释说明用途，例如：  
`nmask: null  # optional real-space mask (.mrc); e.g. mask.mrc`

**验收**：默认 YAML 中无本机绝对路径，新 clone 后可直接用默认配置跑示例（不依赖 mask 时）。

## 二、建议修改（P1）

### 2.1 P2 粒子 I/O：Alignment / Classification 使用只读视图

**依据**：f_012 §4、p_012 §1.5 — 仅在“读取粒子并做对齐/变换/累加”的路径中用只读视图替代 `.data.copy()`，与 reconstruction 一致。

**文件**：
- `pydynamo/src/pydynamo/commands/alignment.py`  
  - 主路径及 `_alignment_cpu_worker` 中粒子加载：`mrc.data.copy().astype(np.float32)` → `vol = np.asarray(mrc.data, dtype=np.float32)`，仅在 `not vol.flags.writeable` 时 `vol = vol.copy()`。
- `pydynamo/src/pydynamo/commands/classification.py`  
  - 主路径及 `_classification_cpu_worker` 中粒子加载：同上；ref 加载可保持 copy（多用途/写缓存），仅粒子热路径改为视图。

**注意**：若下游有 in-place 操作（如 `* mask`），在需要写的那条分支再 copy，避免破坏“只读路径用视图”的语义。

**验收**：纯读取并传入 `align_one_particle` / 变换的粒子路径不无条件全量 copy；与 reconstruction 策略一致。

---

### 2.2 FFT NCC 与实空间 NCC 回归测试

**依据**：f_012 §5、p_012 §2.4 — FFT-based NCC 与 realspace NCC 数值差异在可接受范围内（CC 差异 < 1e-5，shift 差异 ≤ 1 voxel）。

**文件**：`pydynamo/test/test_align.py`（或同目录新建，本规划采用在 test_align.py 中新增用例）。

**修改**：
- 新增测试用例：同一 particle/ref（小体积、无 mask 或全 True mask），分别走 FFT 路径与实空间路径（通过 `_align_single_scale` 或等价入口，保证 cc_mode=ncc、mask 全 True 时走 FFT，否则仅实空间）。
- 断言：CC 差异 < 1e-5；平移 (dx, dy, dz) 各分量差异 ≤ 1 voxel。

**验收**：有明确阈值（CC、shift）的自动化测试，CI 可重复通过，防止后续修改破坏 FFT/实空间一致性。

---

### 2.3 Reconstruction 单线程路径注释

**依据**：f_012 §7 — 逻辑已符合 P2，仅需在注释中写明“只读视图 + 按需 copy，与 P2 粒子 I/O 策略一致”。

**文件**：`pydynamo/src/pydynamo/commands/reconstruction.py`，单线程粒子加载处（约 324–334、403–413 行附近）。

**修改**：在粒子加载块上方或行内增加简短注释，例如：  
“# Read-only view + copy only when not writeable (P2 particle I/O strategy).”

**验收**：注释存在且与实现一致，便于后续维护。
