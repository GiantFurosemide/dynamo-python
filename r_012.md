# f_012: jg_012 实施进度与结果汇总

基于架构规划 [jg_012.md](jg_012.md) 与细化方案 [p_012.md](p_012.md)，实施与测试结果汇总如下。

---

## 1. 已完成的实现

### 1.1 P0：Alignment / Classification CPU 多进程并行

- **alignment**
  - [pydynamo/src/pydynamo/commands/alignment.py](pydynamo/src/pydynamo/commands/alignment.py)：新增 `num_workers` 配置；当 `resolved_device == "cpu"` 且 `num_workers > 1` 时使用 `ProcessPoolExecutor`，模块级 worker `_alignment_cpu_worker(payload)` 在子进程中按 config 路径加载 ref/mask，执行单粒子对齐并返回 `(i, row, tr)`；主进程汇总 rows 与 average。
  - [pydynamo/config/alignment_defaults.yaml](pydynamo/config/alignment_defaults.yaml)：增加 `num_workers: 1` 及注释。
- **classification**
  - [pydynamo/src/pydynamo/commands/classification.py](pydynamo/src/pydynamo/commands/classification.py)：同样增加 `num_workers`；CPU 且 `num_workers > 1` 时使用 `ProcessPoolExecutor`，worker `_classification_cpu_worker(payload)` 使用进程内缓存加载多 ref/mask/wedge，执行多 ref 对齐并返回 `(best_ref, best_row, tr)`。
  - [pydynamo/config/classification_defaults.yaml](pydynamo/config/classification_defaults.yaml)：增加 `num_workers: 1`。
- **runtime**
  - [pydynamo/src/pydynamo/runtime.py](pydynamo/src/pydynamo/runtime.py)：新增 `resolve_cpu_workers(config_value, default=1)`，`<=0` 时返回 `max(1, cpu_count - 1)`，供 alignment/classification/reconstruction 共用。

### 1.2 P0：Classification 每轮迭代粒子只读一次

- [pydynamo/src/pydynamo/commands/classification.py](pydynamo/src/pydynamo/commands/classification.py)：
  - 对齐阶段（单线程、多 GPU、多进程 CPU）均改为返回 `(best_ref, best_row, tr)`，其中 `tr` 为已变换体积；主进程按 ref 累加到 `per_ref_acc` / `per_ref_used`。
  - 平均阶段不再按粒子路径重新读 MRC，直接使用 `per_ref_acc[r] / per_ref_used[r]` 写 average；仅从 JSONL 写 tbl，不再 open 粒子文件。

### 1.3 P1：Alignment 平移搜索 FFT-based NCC

- [pydynamo/src/pydynamo/core/align.py](pydynamo/src/pydynamo/core/align.py)：
  - 新增 `_ncc_volume_fft(particle_eval, ref_rot, mask)`：当 mask 为 None 或全 True 且 `cc_mode == "ncc"` 时，用 FFT 一次性计算整幅 3D NCC 体积，shift (0,0,0) 位于体积中心。
  - 在 `_align_single_scale` 中，对每个角度若 `use_fft_ncc` 则先算 `ncc_vol`，内层 shift 循环改为在 `ncc_vol` 上按 (dx,dy,dz) 索引取值，避免逐平移 `shift()` + NCC；其他 cc_mode 或非全 mask 仍走原有实空间循环。

### 1.4 P1：Reconstruction 多进程按块累加

- [pydynamo/src/pydynamo/commands/reconstruction.py](pydynamo/src/pydynamo/commands/reconstruction.py)：
  - 新增 `recon_workers` 配置与 `_reconstruction_chunk_worker(payload)`；按粒子块划分任务，worker 内加载 mask/wedge、逐粒子读 MRC、变换并累加，返回 `(acc_or_fft_sum, n_acc)`。
  - Fourier 路径与 realspace 路径在 `recon_workers > 1` 时均使用 `ProcessPoolExecutor` 提交块任务，主进程合并 partial sum 与 count。
  - [pydynamo/config/reconstruction_defaults.yaml](pydynamo/config/reconstruction_defaults.yaml)：增加 `recon_workers: 1`。

### 1.5 P2：粒子 I/O 只读视图（mmap 友好）

- [pydynamo/src/pydynamo/commands/reconstruction.py](pydynamo/src/pydynamo/commands/reconstruction.py)：粒子加载改为 `vol = np.asarray(mrc.data, dtype=np.float32)`，仅在 `not vol.flags.writeable` 时再 `vol = vol.copy()`，减少不必要的全量拷贝（主循环与 chunk worker 均已修改）。

### 1.6 P2：Crop 与文档

- Crop 已有 `num_workers` 与线程池，未改逻辑；[pydynamo/README.md](pydynamo/README.md) 已补充 alignment/classification 的 `num_workers` 与 reconstruction 的 `recon_workers` 说明。

## 3. 未实现 / 后续可做

- **GPU 路径**：未改；仍为单粒子/单角度调度，未做批量角度/批量粒子 kernel。
- **Alignment 角度预旋转缓存 / 预计算 FFT**：未做；仅实现了平移侧 FFT NCC。
- **Crop 异步写队列**：未做；仍为现有线程并行与同步写。
- **Benchmark 脚本扩展**：未在本次增加多 worker profile；可在 benchmark_profiles.py 中按 p_012 测试方案补上。

---

## 4. 配置与兼容性

- 默认 `num_workers=1`（alignment/classification）、`recon_workers=1`（reconstruction），与原有单进程行为一致。
- 多进程采用「worker 内按 config 路径加载 ref/mask」或进程内缓存，避免大对象序列化；兼容 spawn（如 Windows）与 fork。
