## 二、实现核对详情

### 2.1 README（§2.1）

- **位置**：`pydynamo/README.md` §5 下「Recommended settings for ~60k particles」。
- **核对**：表格含 alignment num_workers 8–16、multigrid_levels 2、cone_step/inplane_step 15、reconstruction recon_workers 8–16、classification num_workers 8–16、crop num_workers 4–16；说明中已写 num_workers=0（auto）在 60k 时建议显式 8–16、recon_workers 过大增加调度与合并成本、crop 不宜超过 tomogram 数×每 tomo 并行数或 CPU 核数。
- **结论**：符合 p_015 §2.1 及 jg_015 §4 推荐配置表。

### 2.2 Alignment Ref/Mask 共享（§2.2）

- **位置**：`pydynamo/src/pydynamo/commands/alignment.py`。
- **核对**：
  - 模块级全局 `_worker_ref`, `_worker_mask`, `_worker_wedge`, `_worker_params` 及 `_alignment_worker_init(ref_vol, align_mask, wedge_mask, align_params)` 已实现（约 31–44 行）。
  - `_alignment_cpu_worker_impl(payload)` 仅使用 payload 中 `i, p_path, full_path, seed_row`，从全局取 ref/mask/wedge/params，加载粒子后调用 `align_one_particle`，返回 `(i, row, tr)`（约 47–133 行）。
  - 主进程在「CPU 且 num_workers > 1」分支（约 537–579 行）：构建 `align_params`（含 cone_step、tdrot_step、tdrot_range、cone_range、inplane_step、inplane_range、shift_search、lowpass_angstrom、pixel_size、multigrid_levels、shift_mode、subpixel、cc_mode、cc_local_window、cc_local_eps、angle_sampling_mode、wedge_apply_to、fsampling_mode、subpixel_method、wedge_ftype、wedge_ymin/ymax/xmin/xmax）；使用 `ProcessPoolExecutor(..., initializer=_alignment_worker_init, initargs=(ref_vol, align_mask, wedge_mask, align_params))`；提交的 payload 仅 `{"i", "p_path", "full_path", "seed_row"}`，提交给 `_alignment_cpu_worker_impl`。
  - 原 `_alignment_cpu_worker`（每任务内自加载 ref/mask）仍保留，主多进程路径已切到 initializer + `_alignment_cpu_worker_impl`。
- **结论**：符合 p_015 §2.2；多进程时每个 worker 仅通过 initializer 加载一次 ref/mask/wedge/params。

### 2.3 Reconstruction chunk 流式（§2.3）

- **位置**：`pydynamo/src/pydynamo/commands/reconstruction.py`，`_reconstruction_chunk_worker` docstring（约 45–51 行）。
- **核对**：已写明「Chunk is processed in a streaming way: read one particle, transform, add to accumulator, discard (no full-chunk load in memory; jg_015 §3.2)」。
- **结论**：符合 p_015 §2.3，与实现一致。

### 2.4 Benchmark 60k profile（§2.4）

- **位置**：`pydynamo/src/pydynamo/scripts/benchmark_profiles.py`。
- **核对**：`PROFILE_DEFAULTS` 含 `60k_mp4`（particles=1000, box=48, num_workers=4）、`60k_mp8`（particles=1000, box=48, num_workers=8）；模块 docstring 含「60k-scale (for CI/regression): --profiles 60k_mp4,60k_mp8 (1000 particles, multi-worker)」；`_run_profile` 对 `num_workers` 的调度会正确使用 4/8 workers。
- **结论**：符合 p_015 §2.4；`--profiles small,60k_mp4` 或 `60k_mp8` 可运行并输出 JSON。

### 2.5 Crop num_workers 说明（§2.5）

- **位置**：`pydynamo/README.md`「Recommended settings for ~60k particles」表格与说明。
- **核对**：crop 行推荐 4–16；说明中已写「If the number of tomograms is much smaller than num_workers, some workers may sit idle; keep num_workers at or below roughly (tomogram count × desired per-tomogram parallelism) or CPU count」。
- **结论**：符合 p_015 §2.5。

## 三、建议修改（可选）

### 3.1 Benchmark 60k profile 的「非完整 60k」说明（可选）

**依据**：p_015 §2.4 要求「在脚本 docstring 或注释中说明：60k 量级 profile 用 1000–2000 颗粒 + 多 worker 模拟，便于 CI/回归；完整 60k 端到端计时可由用户本地运行」。

**现状**：docstring 已有「60k-scale (for CI/regression): --profiles 60k_mp4,60k_mp8 (1000 particles, multi-worker)」，未显式写「完整 60k 端到端由用户本地运行」。

**建议**：在 docstring 或 PROFILE_DEFAULTS 上方注释中补一句，例如：  
「Full 60k end-to-end timing can be run locally by the user; these profiles use 1000 particles + multi-worker for CI/regression.」

**标准**：读者能区分「60k 量级回归/容量规划用 1000 颗粒」与「真实 60k 端到端需用户自行跑」即可。
