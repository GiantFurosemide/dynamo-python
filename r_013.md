- **结果**：满足 f_012 §5、p_012 §2.4 的阈值要求；新增对 `_ncc_volume_fft` 的导入。

## 三、建议修改（P2）— 已完成

### 3.1 Benchmark 脚本多 worker profile

- **文件**：`pydynamo/src/pydynamo/scripts/benchmark_profiles.py`
- **修改**：
  - 模块 docstring 补充：说明多进程 alignment 与多 worker reconstruction 的用法及输出含义。
  - 新增 profile：`small_mp2`、`small_mp4`（多进程 alignment，num_workers=2/4）、`recon_mp2`（多 worker 重建，recon_workers=2）。
  - 多进程 alignment：`_align_worker_init` / `_align_worker_task`，使用 `ProcessPoolExecutor` 与 initializer 传入 ref/mask，按粒子并行执行 `align_one_particle`，输出 elapsed_s、throughput_particles_per_s、peak_rss_mb 及 num_workers。
  - 多 worker 重建：`_recon_chunk_worker` / `_run_recon_profile`，按 chunk 使用 `ProcessPoolExecutor` 执行 `apply_inverse_transform` + 累加，worker 内按 seed 生成体积避免大数组序列化，输出含 recon_workers。
- **结果**：脚本可运行；`--profiles small,small_mp2,small_mp4,recon_mp2` 可输出可解析的 JSON 指标。运行方式见脚本 docstring 及 `--help`。
