# pydynamo 性能优化实施计划（p_019）

基于架构规划 jg_019.md，本计划详述各优化项的修改步骤与验收标准。

---

## P0.1 GPU 网格计算迁移

**目标**：将 `_rotate_volume_torch_gpu_batch` 中的网格计算从 CPU/NumPy 迁移到 GPU/PyTorch。

**修改文件**：`pydynamo/src/pydynamo/core/align.py`

**步骤**：
1. 新增 `_euler_zxz_to_rotation_matrix_batch(triplets, device)`：在 GPU 上批量计算 ZXZ 欧拉角对应的 3×3 旋转矩阵，返回 `[B, 3, 3]`。
2. 修改 `_rotate_volume_torch_gpu_batch`：
   - 将 `triplets` 转为 GPU tensor `[B, 3]`（tdrot, tilt, narot 度）
   - 调用 `_euler_zxz_to_rotation_matrix_batch` 得到 `rot_mats [B, 3, 3]`
   - 在 GPU 上生成坐标网格 `[d, h, w, 3]`（中心化后）
   - 用 `torch.bmm(rot_mats, coords_flat)` 得到旋转后坐标，再转为 grid_sample 格式
   - 调用 `F.grid_sample` 与现有逻辑一致

**验收**：`test_align.py` 中 GPU 相关测试通过；与修改前 `_rotate_volume_torch_gpu_batch` 输出数值一致（允许 1e-5 浮点误差）。

## P0.2 批量 FFT NCC

**目标**：新增 `_ncc_volume_fft_torch_batch`，支持对 `[B, D, H, W]` 的 ref_rot_batch 一次性计算 B 个 NCC 体积。

**修改文件**：`pydynamo/src/pydynamo/core/align.py`

**步骤**：
1. 新增 `_ncc_volume_fft_torch_batch(part_eval_t, ref_rot_batch)`：
   - `part_eval_t`: `[D, H, W]`，`ref_rot_batch`: `[B, D, H, W]`
   - 将 `part_eval_t` 扩展为 `[B, D, H, W]`（复制 B 份）或保持单体积，用 `torch.fft.fftn` 对 batch 做 FFT
   - 计算互相关：`ifftn(conj(fa) * fb)`，其中 fa 为 part 的 FFT（可 broadcast），fb 为 ref_rot_batch 的 FFT
   - 归一化：每个 b 独立计算 sigma_a、sigma_b（part 相同，ref 每 b 不同）
   - 返回 `[B, D, H, W]` 的 ncc_vol

2. 修改 `_align_single_scale_torch_gpu` 的 batch 分支（`batch_size > 1` 且 `use_fft_ncc`）：
   - 用 `_ncc_volume_fft_torch_batch(part_eval_t, ref_rot_batch)` 替代 per-angle 循环
   - 对返回的 `ncc_vol_batch [B, D, H, W]`，逐 b 在 `shift_search_mask_t` 内取 argmax，得到 B 个 (dx, dy, dz, cc)
   - 取 B 个中的 best 作为 best_params

**验收**：GPU 全 mask 路径与修改前对齐结果一致；`test_align.py` 通过。

**f_019 F1.1 修正**：`fb` 需对零均值 `b` 做 FFT，与单体积 NCC 一致。

## P1.1 Classification GPU swap ref 并行

**目标**：当 `device=cuda` 且 `swap=True` 且 `nref>1` 时，在 `_align_particle_task` 内用 ThreadPoolExecutor 并行对齐到各 ref。

**修改文件**：`pydynamo/src/pydynamo/commands/classification.py`

**步骤**：
1. 在 `_align_particle_task` 内，当 `swap and nref > 1` 且 `resolved_device == "cuda"` 时：
   - 用 `ThreadPoolExecutor(max_workers=min(nref, 8))` 执行 `_align_ref(r)`，其中 `_align_ref` 调用 `align_one_particle(..., device=resolved_device, device_id=local_device_id)`
   - 注意：多 GPU 时每个 ref 可分配不同 device_id，或同一 GPU 时多线程共享
   - 主线程汇总 `results` 中的 best_cc、best_ref、best_row

2. 保持 `swap=False` 或 `nref==1` 时的原有串行逻辑不变。

**验收**：`test_classification.py` 通过；swap=True 多 ref 时行为与 CPU chunk 路径一致（分类结果等价）。

## P1.2 GPU 路径预取

**目标**：在 alignment 和 classification 的 GPU 分支增加预取队列，后台预加载 2–4 个颗粒。

**修改文件**：
- `pydynamo/src/pydynamo/commands/alignment.py`
- `pydynamo/src/pydynamo/commands/classification.py`

**步骤**：

**Alignment**：
1. 在 GPU 分支（`resolved_device == "cuda"`）中，若 `len(tasks) > 1`：
   - 增加 `prefetch_queue_size = max(0, int(config.get("prefetch_queue_size", 2)))`
   - 若 `prefetch_queue_size > 0`：创建 `queue.Queue(maxsize=min(prefetch_queue_size, len(tasks)))`
   - 后台线程 `_prefetcher`：遍历 tasks，对每个 task 执行 `mrcfile.open` + `np.asarray`，将 `(i, p_path, full_path, part, seed_row)` 放入队列
   - 主循环从队列 `get()` 消费，调用 `_run_one` 时传入已加载的 part，避免 `_run_one` 内重复读盘
   - 需修改 `_run_one` 签名：支持可选的 `part_preloaded` 参数，若提供则跳过读盘

2. 注意：当前 GPU 路径 `_run_one` 是闭包内定义的，接收 `task_tuple`，内部自己读盘。需重构为：
   - 方案 A：预取线程放入 `(task_tuple, part_ndarray)`，主循环调用 `align_one_particle(part_ndarray, ...)` 并手动处理 row 构建
   - 方案 B：保持 `_run_one(task_tuple)` 但增加 `part_override` 可选参数，若提供则用 part_override 替代读盘

**Classification**：
1. 在 GPU 分支（`resolved_device == "cuda"`）的迭代循环内，对 `path_tuples`：
   - 若 `prefetch_queue_size > 0`：创建预取队列，后台线程预加载 `(pidx, p_path, part)` 放入队列
   - 主循环从队列消费，调用 `_align_particle_task` 时传入已加载的 part
   - 需修改 `_align_particle_task` 签名：增加可选 `part_preloaded`，若提供则跳过读盘

2. 注意：`_align_particle_task` 当前接收 `(pidx, p_path)`，内部通过 `_resolve_particle_path` 和 `mrcfile.open` 读盘。需改为支持 `part_preloaded` 可选参数。

**验收**：现有测试通过；预取为可选，`prefetch_queue_size=0` 时行为与修改前一致。
