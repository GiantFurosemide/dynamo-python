# pydynamo 性能优化实施规划（p_016）

基于架构文档 jg_016.md 的性能卡点分析，本规划细化实施步骤、文件修改点与验收标准。

---

## 1. 目标与范围

| 目标 | 说明 |
|------|------|
| 降低 IPC 传输量 | 60k 颗粒时从 ~60 GB 降至 ~数百 MB |
| 减少调度开销 | submit/result 次数从 60k 降至 300–1200 |
| 共享 ref/mask | Classification 使用 initializer，消除多 worker 重复读盘 |
| 保持兼容 | 不破坏现有单 worker、GPU 路径、流式 tbl 行为 |

---

## 2. 阶段一：P0 高优先级

### 2.1 Alignment Chunk 提交

#### 2.1.1 设计

- **Chunk 定义**：将颗粒任务按 `chunk_size`（默认 100）打成块，每块包含 `(indices, paths_chunk, seed_rows_chunk)`。
- **Worker 行为**：`_alignment_chunk_worker_impl(payload)` 接收一块，循环处理每颗粒 → 对齐 → 局部累加 `local_acc += tr` → 丢弃颗粒；返回 `(rows_list, local_acc)`。
- **主进程行为**：按 chunk 提交；对每个 result 执行 `avg_acc += local_acc`，`rows_pairs.extend(...)`；最后按 index 排序 rows。

#### 2.1.2 文件与修改点

| 文件 | 修改内容 |
|------|----------|
| `pydynamo/src/pydynamo/commands/alignment.py` | 新增 `_alignment_chunk_worker_impl`；在 CPU 多进程分支中按 chunk 构建 payload、使用 chunk worker；主进程合并 rows 与 avg_acc |
| `pydynamo/config/alignment_defaults.yaml` | 新增 `chunk_size: 100`（可选，若通过 config 读取） |
| `pydynamo/config/synthetic_align*.yaml` | 可选添加 `chunk_size` 示例 |

#### 2.1.3 实现细节

1. **Chunk 构建**：
   ```python
   chunk_size = max(1, int(config.get("chunk_size", 100)))
   chunks = []
   for start in range(0, len(tasks), chunk_size):
       end = min(start + chunk_size, len(tasks))
       chunk_tasks = tasks[start:end]
       indices = [t[0] for t in chunk_tasks]
       paths_chunk = [(t[1], t[2]) for t in chunk_tasks]  # p_path, full_path
       seed_rows_chunk = [tbl_df.iloc[i].to_dict() if i < len(tbl_df) else {} for i in indices]
       chunks.append({"indices": indices, "paths_chunk": paths_chunk, "seed_rows_chunk": seed_rows_chunk})
   ```

2. **Chunk worker 返回值**：
   - `rows_list`: `[(i, row), ...]` 保持与 indices 一致顺序
   - `local_acc`: `np.float64` 与 ref_vol 同 shape

3. **Fallback**：当 `chunk_size >= len(tasks)` 或 `num_workers == 1` 时，可继续使用单颗粒逻辑以简化；或统一使用 chunk（chunk_size=1 等价于原逻辑）。

4. **tbl 流式**：chunk 模式下暂不逐行流式写 tbl，改为收集所有 rows 后按 index 排序再写，避免乱序。若需保持流式，可后续扩展。

#### 2.1.4 验收标准

- [ ] 60k 颗粒、8 workers、chunk_size=100 时，IPC 传输量显著下降（可通过日志或 profiling 验证）
- [ ] 输出 tbl/star、average 与单颗粒模式数值一致（回归测试）
- [ ] `chunk_size=1` 时行为与当前实现等价

### 2.2 Classification Initializer + Chunk

#### 2.2.1 设计

- **Initializer**：`_classification_worker_init(ref_vols, align_mask, wedge_mask, align_params)`，与 alignment 类似，主进程加载后通过 `ProcessPoolExecutor(initializer=..., initargs=...)` 注入。
- **Chunk worker**：`_classification_chunk_worker_impl(payload)` 接收 `(indices, paths_chunk, seed_rows_chunk, swap, ref_to_align_chunk)`，处理多颗粒，返回 `(rows_list, per_ref_local_acc_list)`。
  - `rows_list`: `[(best_ref, best_row), ...]`，best_row 含 ref、rlnImageName 等
  - `per_ref_local_acc_list`: 长度为 nref 的 list，每项为 `np.float64` 累加器
- **主进程**：合并 `per_ref_acc[r] += per_ref_local_acc[r]`，合并 rows 并写入 row_fhs。

#### 2.2.2 文件与修改点

| 文件 | 修改内容 |
|------|----------|
| `pydynamo/src/pydynamo/commands/classification.py` | 新增 `_classification_worker_init`；新增 `_classification_chunk_worker_impl`；移除/替换 `_classification_worker_cache`；CPU 多进程分支使用 initializer + chunk 提交 |
| `pydynamo/config/classification_defaults.yaml` | 新增 `chunk_size: 100` |
| `pydynamo/config/synthetic_class.yaml` | 可选添加 `chunk_size` |

#### 2.2.3 实现细节

1. **Initializer**：
   ```python
   _worker_ref_vols = None
   _worker_align_mask = None
   _worker_wedge_mask = None
   _worker_params = None

   def _classification_worker_init(ref_vols, align_mask, wedge_mask, params):
       global _worker_ref_vols, _worker_align_mask, _worker_wedge_mask, _worker_params
       _worker_ref_vols = ref_vols
       _worker_align_mask = align_mask
       _worker_wedge_mask = wedge_mask
       _worker_params = params
   ```

2. **Chunk payload**：
   - `indices`: 颗粒索引
   - `paths_chunk`: `[(p_path, full_path), ...]`
   - `seed_rows_chunk`: 每颗粒的 seed_row
   - `swap`: bool
   - `ref_to_align_chunk`: `[ref_to_align or None, ...]` 每颗粒的 ref_to_align（swap=False 时）

3. **per_ref_local_acc**：worker 内初始化 `[np.zeros_like(ref_vols[0], dtype=np.float64) for _ in range(nref)]`，每颗粒对齐后 `per_ref_local_acc[best_ref] += tr`。

4. **row_fhs 写入**：主进程收到 chunk 的 rows_list 后，按 best_ref 写入对应 row_fh，与当前逻辑一致。

#### 2.2.4 验收标准

- [ ] 多 worker 时不再每个 worker 独立加载 ref（内存与 I/O 降低）
- [ ] 输出 per-ref averages、tables 与当前实现数值一致
- [ ] `chunk_size=1` 时行为等价
