## 1. 完成项

### 1.1 文档与推荐配置（§2.1）

- **文件**：`pydynamo/README.md`
- **变更**：在「Runtime and parallel defaults」下新增小节 **Recommended settings for ~60k particles**：
  - 表格：alignment num_workers 8–16、multigrid_levels 2、cone_step/inplane_step 15、reconstruction recon_workers 8–16、classification num_workers 8–16、crop num_workers 4–16 等推荐范围。
  - 说明：60k 规模建议显式设置 num_workers/recon_workers 为 8–16，避免 auto 占满机器；recon_workers 过大增加调度与合并成本；crop 的 num_workers 不宜超过「tomogram 数 × 每 tomo 并行数」或 CPU 核数。

### 1.2 Alignment CPU 多进程：Ref/Mask 共享（§2.2）

- **文件**：`pydynamo/src/pydynamo/commands/alignment.py`
- **变更**：
  - 新增模块级全局 `_worker_ref`, `_worker_mask`, `_worker_wedge`, `_worker_params` 及 **`_alignment_worker_init(ref_vol, align_mask, wedge_mask, align_params)`**，在 worker 进程内设置上述全局，使每个 worker 只加载一次 ref/mask/wedge。
  - 新增 **`_alignment_cpu_worker_impl(payload)`**：payload 仅含 `i, p_path, full_path, seed_row`；从全局读取 ref/mask/wedge/params；加载粒子 MRC；调用 `align_one_particle`；返回 `(i, row, tr)`。
  - 主进程在「CPU 且 num_workers > 1」分支中：构建 `align_params`（含 cone_step、tdrot_step、…、wedge_apply_to、fsampling_mode、subpixel_method 及 wedge_ftype/wedge_ymin 等）；使用 **`ProcessPoolExecutor(initializer=_alignment_worker_init, initargs=(ref_vol, align_mask, wedge_mask, align_params))`**；提交的 payload 仅 `(i, p_path, full_path, seed_row)`；提交给 `_alignment_cpu_worker_impl`。
- **效果**：多进程 alignment 时每个 worker 进程仅加载一次 reference 与 mask，60k 任务下不再每任务加载，降低 I/O 与内存重复占用。原 `_alignment_cpu_worker` 仍保留（单任务内自加载 ref）供兼容，主多进程路径已切到 initializer + `_alignment_cpu_worker_impl`。

### 1.3 Reconstruction chunk 流式说明（§2.3）

- **文件**：`pydynamo/src/pydynamo/commands/reconstruction.py`
- **变更**：在 **`_reconstruction_chunk_worker`** 的 docstring 中明确「Chunk is processed in a streaming way: read one particle, transform, add to accumulator, discard (no full-chunk load in memory; jg_015 §3.2)」。
- **效果**：与现有实现一致（本已逐颗粒读-变换-累加-丢弃），便于审查与后续维护。

### 1.4 Benchmark 60k 量级 profile（§2.4）

- **文件**：`pydynamo/src/pydynamo/scripts/benchmark_profiles.py`
- **变更**：
  - 在 **`PROFILE_DEFAULTS`** 中新增 **`60k_mp4`**（particles=1000, box=48, num_workers=4）、**`60k_mp8`**（particles=1000, box=48, num_workers=8），用于 60k 规模下的多进程 alignment 回归与容量评估。
  - 模块 docstring 中补充：60k-scale 示例 `--profiles 60k_mp4,60k_mp8`。
- **验收**：`python -m pydynamo.scripts.benchmark_profiles --profiles small,60k_mp4`（或 `60k_mp8`）在安装依赖后可运行并输出 JSON。

### 1.5 Crop num_workers 说明（§2.5）

- **文件**：`pydynamo/README.md`
- **变更**：已在「Recommended settings for ~60k particles」表格与说明中覆盖：crop num_workers 4–16；当 tomogram 数远小于 num_workers 时部分 worker 可能闲置，建议 num_workers 不超过 tomogram 数×每 tomo 并行数或 CPU 核数。
