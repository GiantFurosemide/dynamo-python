## F1 必须修改（影响正确性）

### F1.1 `_ncc_volume_fft_torch_batch` 使用未零均值 ref 做 FFT

**位置**：`core/align.py` 第 977–978 行

**问题**：`fb = torch.fft.fftn(ref_rot_batch.float(), dim=(-3, -2, -1))` 使用的是原始 `ref_rot_batch`，而单体积版本 `_ncc_volume_fft_torch` 使用的是零均值 `b = ref_rot_t.float() - torch.mean(ref_rot_t)` 的 FFT。NCC 公式要求对零均值信号做 FFT。

**修改**：改为对零均值 `b` 做 FFT：
```python
b = ref_rot_batch.float() - torch.mean(ref_rot_batch, dim=(1, 2, 3), keepdim=True)
...
fb = torch.fft.fftn(b, dim=(-3, -2, -1))
```

**标准**：与 `_ncc_volume_fft_torch` 的数学定义一致，`test_fft_ncc_vs_realspace_ncc_consistency` 通过。

## F2 建议修改（优化/健壮性）

### F2.1 `_euler_zxz_to_rotation_matrix_batch` 角度转换仍在 CPU

**位置**：`core/align.py` 第 793–799 行

**问题**：`np.deg2rad` 与 `np.stack` 在 CPU 上执行，随后 `torch.as_tensor(rad, device=device)` 再传到 GPU。角度转换可完全在 GPU 上完成，减少一次 CPU–GPU 传输。

**修改**：将角度张量先传到 GPU，再用 `torch.deg2rad` 转换：
```python
triplets_t = torch.as_tensor(np.asarray(triplets, dtype=np.float32), device=device)
if triplets_t.dim() == 1:
    triplets_t = triplets_t.reshape(1, 3)
rad_t = torch.deg2rad(triplets_t)
```

**标准**：与修改前数值一致（误差 < 1e-6），测试通过。

---

### F2.2 Classification 单线程预取路径的 `progress_iter` 行为

**位置**：`commands/classification.py` 单 GPU 预取分支

**问题**：`task_iter = iter(lambda: prefetch_q.get(), None)` 在 `progress_iter` 中迭代时，`total=len(path_tuples)` 正确，但迭代器在收到 `None` 时结束，`progress_iter` 可能对 sentinel 做一次多余迭代。当前实现通过 `if item is None: break` 处理，逻辑正确，但可考虑用更明确的迭代方式以提高可读性。

**标准**：保持当前行为，或改为显式 `while True` 循环，可读性更佳。
