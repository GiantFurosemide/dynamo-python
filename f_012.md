# f_012: pydynamo 性能优化代码审查报告

依据架构规划 [jg_012.md](jg_012.md)、实现方案 [p_012.md](p_012.md) 及进度汇总 [r_012.md](r_012.md)，对当前代码进行严格审查。以下为**必须修改项**、**建议修改项**及**需达到的标准**。

---

## 一、必须修改（Bug / 与方案不符）

### 1. Classification：多路径返回值不一致导致运行时错误

**位置**：`pydynamo/src/pydynamo/commands/classification.py`

**问题**：  
调用方统一按 `(best_ref, best_row, tr)` 三元组解包（见 358、369、381 行），但部分分支只返回两个值，会导致 `ValueError: not enough values to unpack (expected 3, got 2)`。

**需修改处**：

| 行号 | 当前代码 | 应改为 |
|------|----------|--------|
| 93 | `return 0, None` | `return 0, None, None` |
| 95 | `return 0, None` | `return 0, None, None` |
| 353 | `return None, None` | `return None, None, None` |
| 355 | `return None, None` | `return None, None, None` |

**标准**：  
- `_classification_cpu_worker` 与 `_align_particle_task` 的所有 return 必须为 `(best_ref, best_row, tr)` 三元组；失败/跳过时 `tr` 为 `None`。  
- 修改后运行：`pytest pydynamo/test/test_classification.py -v` 应全部通过，且 `num_workers=1` 与 `num_workers>1` 的集成测试不出现解包错误。

---

### 2. Classification worker 文档与实现不一致

**位置**：`pydynamo/src/pydynamo/commands/classification.py` 中 `_classification_cpu_worker` 的 docstring（约 48–52 行）。

**问题**：  
文档写 “Returns (best_ref, best_row) or (0, None) on load failure”，实际返回为 `(best_ref, best_row, tr)` 或 `(0, None, None)`。

**需修改**：  
将 docstring 改为与实现一致，例如：  
“Returns (best_ref, best_row, tr); on load failure or shape mismatch returns (0, None, None).”

**标准**：  
文档描述与所有 return 路径一致，且明确说明 `tr` 为已变换体积或 None。

---

### 3. Alignment 默认配置中不应包含本机绝对路径

**位置**：`pydynamo/config/alignment_defaults.yaml` 第 53 行。

**问题**：  
`nmask` 被设为 `/home/muwang/.../mask.mrc`，默认配置应保持可移植、无本机路径。

**需修改**：  
将 `nmask` 改为 `null` 或注释示例路径，例如：  
`nmask: null  # optional real-space mask (.mrc); e.g. mask.mrc`

**标准**：  
默认 YAML 中不得包含本机绝对路径；新 clone 后可直接用默认配置跑通示例（若示例不依赖 mask）。

## 二、建议修改（与 p_012 / jg_012 一致或可维护性）

### 4. P2 粒子 I/O：Alignment / Classification 未使用只读视图

**依据**：  
p_012 §1.5 要求“在所有仅读取粒子并做对齐/变换/累加的路径中”用只读视图替代 `.data.copy()`；r_012 仅写明 reconstruction 已改，alignment/classification 仍为 copy。

**位置**：  
- `pydynamo/src/pydynamo/commands/alignment.py`：主路径与 `_alignment_cpu_worker` 中 `mrc.data.copy().astype(np.float32)`（约 83、313、238 行及 worker 内 61、80 行附近）。  
- `pydynamo/src/pydynamo/commands/classification.py`：主路径与 `_classification_cpu_worker` 中同类用法（约 291、351、73、92 行等）。

**建议**：  
与 reconstruction 一致：  
- 先用 `vol = np.asarray(mrc.data, dtype=np.float32)`（或只读视图）；  
- 仅在对该数组有写操作或下游要求可写时再 `vol = vol.copy()`。  
注意：alignment/classification 中若后续有 in-place 运算（如 `* mask`），应在需要写的那条分支再 copy，避免破坏“只读路径用视图”的语义。

**标准**：  
- 纯读取并传入 `align_one_particle` / 变换的路径，不无条件全量 copy；  
- 若存在 mmap/memmap，多进程 fork 后只读页可共享，降低内存与 I/O（与 jg_012 §5、p_012 §1.5 一致）。

---

### 5. FFT NCC 与实空间 NCC 的回归测试

**依据**：  
p_012 §2.4：“FFT-based NCC 与 realspace NCC 的数值差异在测试中限定在可接受范围（如 CC 差异 < 1e-5，shift 差异 <= 1 voxel）”。

**现状**：  
未发现针对 `_ncc_volume_fft` 与实空间 NCC 的对比测试。

**建议**：  
在 `pydynamo/test/test_align.py`（或新建）中增加回归测试：  
- 同一 particle/ref（可小体积、无 mask 或全 True mask），分别走 FFT 路径与实空间路径；  
- 断言：CC 差异 < 1e-5，平移 (dx,dy,dz) 差异每个分量 ≤ 1 voxel。

**标准**：  
- 有明确阈值（CC、shift）的自动化测试；  
- 默认/CI 下可重复通过，防止后续修改破坏 FFT/实空间一致性。

---

### 6. Benchmark 脚本扩展（p_012 §2.3）

**依据**：  
p_012 测试方案：“扩展 benchmark_profiles.py：增加 profile「多进程 alignment」（如 num_workers=2,4）、「多 worker reconstruction」”。

**现状**：  
r_012 写明“本次未修改该脚本，可作为后续步骤”。

**建议**：  
在 `pydynamo/src/pydynamo/scripts/benchmark_profiles.py` 中增加：  
- 多进程 alignment（如 num_workers=2, 4）的 profile；  
- 多 worker reconstruction（如 recon_workers=2）的 profile；  
- 输出吞吐或耗时到 JSON/日志，便于回归与容量规划。

**标准**：  
- 脚本可运行且输出可解析的指标；  
- 文档或注释说明如何运行与解读结果。

---

### 7. Reconstruction 单线程路径的粒子读取方式

**位置**：`pydynamo/src/pydynamo/commands/reconstruction.py` 单线程分支（约 324–334、403–413 行）。

**现状**：  
已使用 `np.asarray(mrc.data, dtype=np.float32)` 并在 `not vol.flags.writeable` 时 copy，与 P2 一致。

**建议**：  
无需改逻辑；仅建议在注释中写明“只读视图 + 按需 copy，与 P2 粒子 I/O 策略一致”，便于后续维护。

## 三、需达到的总体标准

1. **兼容性**  
   - `num_workers=1`、`recon_workers=1` 时行为与优化前一致；  
   - 默认配置无本机绝对路径，可在新环境直接使用。

2. **正确性**  
   - 所有对齐/分类/重建的 return 与调用方解包一致，无 “not enough values to unpack”；  
   - FFT NCC 与实空间 NCC 在约定阈值内一致（有对应回归测试）。

3. **与规划一致**  
   - P0：CPU 多进程、classification 单读累加 —— 已实现，仅需修上述 bug；  
   - P1：FFT 平移 NCC、reconstruction 多进程 —— 已实现；  
   - P2：粒子 I/O 只读视图 —— reconstruction 已做，alignment/classification 建议按 §4 补全；  
   - 文档与注释与实现一致（含 worker 返回值、配置项含义）。

4. **测试与可观测性**  
   - 现有 pytest（含 test_runtime、test_alignment_command、test_classification、test_reconstruction_command、test_align）全部通过；  
   - 建议增加：FFT vs 实空间 NCC 回归测试、benchmark 多 worker profile。
