## 二、建议修改 / 改进

### 2.1 FFT NCC 回归测试：平移差异断言（可选）

**依据**：p_013 §2.2 要求「CC 差异 < 1e-5；**平移 (dx, dy, dz) 各分量差异 ≤ 1 voxel**」。当前测试仅验证：在 FFT 峰值对应平移处，实空间 NCC 与 FFT NCC 的 CC 值一致（< 1e-5），**未**验证「整轮对齐 FFT 路径 vs 实空间路径得到的 (dx, dy, dz) 各分量差 ≤ 1 voxel」。

**位置**：`pydynamo/test/test_align.py`，`test_fft_ncc_vs_realspace_ncc_consistency`（约 679–703 行）。

**建议**（可选）：  
若需严格满足 p_013 对「平移差异」的表述，可增加：  
- 使用相同 particle/ref、全 True mask，分别走**启用 FFT NCC** 与**仅实空间 NCC** 的 `_align_single_scale`（或等价入口，需能区分两种路径，例如通过 mask 或内部开关）；  
- 断言两次输出的 (dx, dy, dz) 各分量差 ≤ 1 voxel。  
若当前不暴露「仅实空间」路径，可暂不实现，在文档或测试 docstring 中注明「当前仅验证同 shift 下 CC 一致；平移输出一致性可后续补充」。

**标准**：  
- 若实现：有明确断言 `max(|dx_fft - dx_rs|, |dy_fft - dy_rs|, |dz_fft - dz_rs|) <= 1`；  
- 若不实现：测试 docstring 或注释中说明仅覆盖 CC 一致性，平移一致性为后续改进。

---

### 2.2 Benchmark：recon_mp2 与真实 reconstruction 的差异说明（可选）

**现状**：`_run_recon_profile` 在 worker 内用 `rng.standard_normal` 生成体积并做 `apply_inverse_transform` + 累加，不读 MRC，与真实 reconstruction 的 I/O 与数据路径不同。

**建议**：在 `benchmark_profiles.py` 的模块 docstring 或 `_run_recon_profile` 的 docstring 中增加一句说明，例如：  
“recon_mp2 measures transform+sum throughput with synthetic in-memory volumes; it does not include MRC I/O.”

**标准**：读者能区分「纯计算 throughput」与「含 I/O 的 reconstruction」的差异，避免误读为真实重建耗时。
