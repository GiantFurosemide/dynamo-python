## 实施详情

### P0.1 CPU rotate_volume Numba
- `core/align.py`: 新增 `_rotate_volume_numba`、`_rotate_volume_scipy`，`rotate_volume` 默认用 Numba（可用时）
- `pyproject.toml`: 新增 `perf = ["numba>=0.56"]` 可选依赖
- `test_align.py`: 新增 `test_rotate_volume_numba_matches_scipy`

### P0.2 GPU FFT NCC for shifts
- `core/align.py`: 新增 `_ncc_volume_fft_torch`，`_align_single_scale_torch_gpu` 在 mask 全 1 时用 FFT 一次性得 ncc_vol，替代逐 shift 循环
- **f_018 F2**: 新增 `_build_shift_search_mask_torch`，用 mask+argmax 替代 `for sx,sy,sz in shifts` 循环，达成 O(1) shift 搜索

### P1.1 Chunk 内预取
- `alignment.py` / `classification.py`: 新增 `prefetch_queue_size`，worker 内预取线程 + queue
- `alignment_defaults.yaml` / `classification_defaults.yaml`: prefetch_queue_size: 2

### P1.2 Classification ref 级并行
- `classification.py`: `_classification_chunk_worker_impl` 中 swap=True 且 nref>1 时用 ThreadPoolExecutor 并行 align_one_particle

### P1.3 路径排序
- `alignment.py`: tasks 按 full_path 排序
- `classification.py`: path_tuples 按 full_path 排序后用于 chunk/迭代

### P2.1 chunk_size auto
- `alignment.py` / `classification.py`: chunk_size 为 0/auto 时，`chunk_size = max(50, min(200, n // (num_workers*4)))`

### P2.3 文档化
- `README.md`: 60k 推荐表增加 chunk_size、prefetch_queue_size、cone_step 探索建议；chunk tbl 非流式说明；I/O 瓶颈时减少 num_workers

### f_018 审计修改

- **F1**: `test_fft_ncc_vs_realspace_ncc_consistency` 放宽容差至 0.05，注释说明 FFT 使用全局 ref mean/std，realspace 使用 ref_shifted（零填充边界）mean/std，两者在边界 shift 下存在已知差异
- **F2**: GPU FFT NCC 路径改用 `_build_shift_search_mask_torch` 构建有效 shift 区域 mask，`ncc_masked[~mask]=-2` 后 `argmax`，直接得到最佳 (dx,dy,dz)，不再循环 shifts
- **F3**: `alignment_defaults.yaml` 增加 `coarse_cone_step`、`coarse_inplane_step` 注释，配置预留；multigrid 粗阶段使用该配置的实现待后续
